# Few-Shot Examples — Quentar Studio cold outreach (V5)

После третьего раунда правок менеджера структура сообщения теперь
**6 частей**, добавлено поле `outcome` — конкретный сценарий что
изменится после установки сайта (UX клиентки + облегчение владельца).

Ключевые принципы:
- Один источник наблюдения (либо 2ГИС, либо Instagram)
- Конкретный комплимент глядя на их работы
- Одна дыра, без двойных упоминаний
- Без цены, без срока, без «5 минут», без «мы делаем», без «простой»
- **outcome — самое важное поле**. Показывайте функционал глазами клиентки
  плюс облегчение владельца. БЕЗ выдуманных историй ДО решения

Длина 60-100 слов sweet spot 70-90.

---

## Example 1 — phone tier, nail studio, Бишкек (с именем владельца)

**Input:**
```
business_name: Parizat nail studio
city: Бишкек
niche: маникюр / косметология
owner_name: Паризат
phone: +996509951195
company_instagram: @parizat.nailstudio
channel: WhatsApp
```

**Output:**
```json
{
  "hook": "Паризат, здравствуйте 💅",
  "observation": "Зашёл в @parizat.nailstudio. Работы аккуратные, видно мастерство.",
  "problem": "Прайса в постах не нашёл. Наверное в директ часто спрашивают цены?",
  "value": "Делаем компактные сайты под bio Instagram: прайс, мастера с портфолио, кнопка записи в одной ссылке.",
  "outcome": "Клиентка заходит, сразу видит сколько стоит френч или дизайн, выбирает мастера и записывается на время. Без ожидания ответа в директе. У вас остаются только сложные запросы, типовые вопросы клиентки закрывают сами.",
  "ask": "Скинуть пример как мог бы выглядеть под Parizat?"
}
```

83 слова. outcome даёт уровень 1 (UX: «клиентка заходит, видит, выбирает,
записывается») плюс уровень 2 (relief: «остаются только сложные запросы»).

## Anti-examples — НЕ так

### ❌ Anti-1: outcome как пустая фраза

```json
{"outcome": "Сайт улучшит ваш бизнес и повысит конверсию."}
```

Это пустая фраза маркетолога. Замените на конкретный сценарий что
увидит клиентка плюс что изменится для владельца.

### ❌ Anti-2: outcome как выдуманная история ДО решения

```json
{"outcome": "Клиентка зашла в Instagram, ищет цены, не находит, уходит к конкуренту с сайтом."}
```

Это вымысел про текущее поведение, не сценарий ПОСЛЕ установки.
Менеджер: «это копипаста под шаблон, мозг включает оборону».

Правильно: «Клиентка заходит на сайт, сразу видит цены, выбирает мастера,
записывается» — это после установки, демонстрация UX.

### ❌ Anti-3: цена и срок в value/outcome

```json
{
  "value": "Делаем сайты от 162 000 ₸ за 7-10 дней.",
  "outcome": "За неделю запустим сайт и клиентка сразу будет видеть всё нужное."
}
```

Цена и срок в первом сообщении превращают диалог в коммерческий до
согласия. Обсудим во втором сообщении.

### ❌ Anti-4: «Мы делаем» + «простой» + em-dash

```json
{
  "value": "Мы делаем простой сайт — лендинг с прайсом — где клиент видит всё."
}
```

«Мы делаем» — реклама компании. «Простой» — лишнее. Em-dash — AI-тэлл.

Правильно: «У нас есть формат: сайт в bio с прайсом, мастерами и записью».

### ❌ Anti-5: «5 минут» + «без обязательств»

```json
{"ask": "Скинуть пример? Минут 5, без обязательств."}
```

Узнаваемая sales-техника маленького обязательства. Заменить на просто
«Скинуть пример?» или «Покажу как мог бы выглядеть».

### ❌ Anti-6: два источника наблюдения

```json
{"observation": "Нашёл вас на 2ГИС в Усенбаева. Зашёл в @parizat.nailstudio, посмотрел работы."}
```

Звучит как отчёт о работе. Упомяните один источник.

### ❌ Anti-7: «Пишу по» в hook

```json
{"hook": "Здравствуйте, пишу по Parizat nail studio"}
```

Call-center формулировка. Замените на «Паризат, здравствуйте» или
«Здравствуйте» (без «пишу по»).
